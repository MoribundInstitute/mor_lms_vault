## đź§© The Unified Stack

The `mor_lms_desktop` crate does not handle heavy logic. It is purely the UI and the Director. It orchestrates the following internal dependencies:

1. **The Brain (`vault_core`)**: The UI calls this crate to calculate SM-2 spaced repetition math and parse flashcard schemas.
2. **The Safe (`vault_crypto`)**: The UI uses this to encrypt/decrypt `my_brain.enc` so local data remains secure at rest.
3. **The Data Pipeline (`mor_mediawiki_miner`)**: Generates offline `smoldata.json.br` shards from our custom MediaWikis. The UI parses these files to render the "Vault Explorer".
4. **The Mailman (`SyncManager`)**: A custom Rust wrapper that directly controls the `syncthing` binary to push data across devices.

## âš™ď¸Ź The Consent-Driven Sync Engine

We refuse to act like modern spyware. Syncthing is integrated using a strict **Consent-Driven Model**:

* **State 0 (App-Bound):** By default, `syncthing` is booted as a child process of the UI. When the user closes the app, the sync engine is violently killed. Zero background RAM usage.
* **State 1 (OS Daemon):** If the user explicitly opts-in via the UI settings, the UI executes systemd commands (`systemctl --user enable syncthing`) to hand the engine over to the Linux OS. Syncing continues 24/7 in the background. If they opt-out, the UI disables the daemon and reclaims the child process.
